# Personal Preferences

## Style

- Be concise.
- Don't add comments unless the logic is non-obvious.
- Don't add docstrings to code you didn't change.
- Prefer simple solutions over clever ones.

## Workflow

- Read existing code before modifying it. Understand patterns already in use.
- Run tests after making changes when a test suite exists.
- When exploring unfamiliar code, use subagents to keep main context clean.
- Use `/compact` proactively when context grows large.
